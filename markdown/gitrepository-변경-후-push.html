🤔 문제상황

코드 작업을 하다가, github 정리좀 한다고
작업중이던 repository의 이름을 변경했다.
그리고나서 commit을 하고 push를 했더니

기존에 remote 되어있는 주소와 달라서 error가 떴다.

😲 해결방법

처음엔 remote 주소를 다시 세팅하는 방법이 있다길래
$ git remote set-url /레포주소
명령어로 시도했으나 주소만 바뀐채 연결이 되지않고 그대로 거절당했다..

알아보니 우선 기존 레포지토리에서 pull로 끌어오고 add하고 push한 뒤
이전 remote 주소를 삭제하고 다시 연결하면 된다길래

$ git pull
$ git add .
$ git push
$ git remote remove origin
$ git remote add origin /주소

이렇게 작업을 마쳤는데도 똑같은 에러를 뿜어냈다.

문제는, origin의 master 브랜치에 있는 내용을 pull해야 했기 때문에
$ git pull => $ git pull origin master
로 바꿔서 작업을 했더니 드디어 성공..!

✋ 한가지 더!
pull 작업을 마쳤더니

commit 메세지를 작성하라는 편집기가 뜬다.
java를 처음 배울때의 기억을 더듬더듬해서..
commit 메세지는 vscode로 작성해서 올리고
ESC키 => :(콜론) => q!입력 => enter
순으로 입력하면 편집기에서 나오게된다.

이후에 다시 push 작업을 하면 완성!
